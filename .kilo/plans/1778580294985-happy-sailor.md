### 4.2 Locking Mechanism for Leave Processing
**File**: Update `leaves/views.py` and `leaves/models.py`
**Logic**:
- When handler opens a leave in "ΠΡΟΣ ΕΠΕΞΕΡΓΑΣΙΑ" status:
  - If not locked: lock it to this handler (set locking_user, locked_at)
  - If locked by same handler: allow continue
  - If locked by different handler: show "Locked by X" message, offer manual unlock
  - Auto-unlock after 30 minutes of inactivity (check in view, not celery for simplicity)
- Show "Υπό επεξεργασία από [handler name]" to other handlers
- Unlock when handler saves/completes/rejects

### 4.3 Revocation System
**File**: Update `leaves/models.py` LeaveRequest methods

**Two types of revocation**:
1. **Ανάκληση Αίτησης** (until ΠΡΟΣ ΕΠΕΞΕΡΓΑΣΙΑ):
   - Available from ΚΑΤΑΧΩΡΗΘΗΚΕ through ΕΓΚΡΙΣΗ ΑΠΟ ΠΡΟΪΣΤΑΜΕΝΟ
   - User clicks button → status changes to ΑΝΑΚΛΗΣΗ ΑΙΤΗΣΗΣ ΑΠΟ ΑΙΤΟΥΝΤΑ
   - No leave days deducted
   - Terminal state

2. **Ανάκληση Ολοκληρωμένης Άδειας** (from ΟΛΟΚΛΗΡΩΜΕΝΗ):
   - User selects completed leave → clicks "Ανάκληση Ολοκληρωμένης"
   - Creates NEW LeaveRequest with `parent_leave` pointing to original
   - Status starts at ΚΑΤΑΧΩΡΗΘΗΚΕ, goes through full workflow
   - PDF includes reference to original decision
   - Original leave status changes to ΑΝΑΚΛΗΣΗ ΟΛΟΚΛΗΡΩΜΕΝΗΣ ΑΔΕΙΑΣ
   - Leave days are restored to user's balance

3. **Μερική Ανάκληση**:
   - Same as above but only for specific days
   - Create new LeaveRequest with `parent_leave`
   - Track revoked_days on original
   - Recalculate remaining balance

4. **Delete in ΠΡΟΣ ΕΠΕΞΕΡΓΑΣΙΑ** (handler only):
   - Only handler can delete
   - Log the deletion
   - Restore leave days to user's balance

### 4.9 SDEY Manager Restrictions
**File**: Update `accounts/models.py` User model and views
**Logic**:
- SDEY managers (users with `department.department_type.code == 'SDEY'`) have `can_request_leave = False`
- They can ONLY view dates and status of their SDEY's leave requests
- Cannot view attachments
- Cannot approve
- Cannot request leave for themselves
- Update all views to check `can_request_leave` before allowing request
**Implementation**:
- Override `can_request_leave` property in User model:
```python
@property
def can_request_leave(self):
    # SDEY managers can never request leave
    if self.department and self.department.department_type and \
       self.department.department_type.code == 'SDEY':
        return False
    # Root department managers (e.g., PDEDE director) cannot request
    if self.department and self.is_manager_of_department(self.department):
        if not self.department.parent_department:
            return False
    return True
```
- In all views that create leave requests, check in `dispatch()`:
```python
if not self.request.user.can_request_leave():
    raise PermissionDenied("Δεν έχετε δικαίωμα υποβολής αιτήσεων άδειας.")
```
- In templates, hide "Νέα Αίτηση" button: `{% if user.can_request_leave %}`
- On registration approval, auto-set `can_request_leave = False` for SDEY users
