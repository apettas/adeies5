# Εγχειρίδιο Χειριστή Αδειών

Οδηγίες για την πλήρη επεξεργασία αιτήσεων από το πρωτόκολλο ΠΔΕΔΕ μέχρι την ολοκλήρωση και τη ροή ΣΗΔΕ.

---

## 1. Σύνδεση και dashboard

1. Συνδεθείτε με λογαριασμό **Χειριστής Αδειών** (ή επιλέξτε «Χειριστής Αδειών» από την εναλλαγή ρόλων).
2. Ανοίγει το **Dashboard Χειριστή** με καρτέλες (tabs) ανά φάση επεξεργασίας.

### Κύριες καρτέλες

| Καρτέλα | Τι περιέχει |
|---------|-------------|
| **Για Πρωτόκολλο** | Αιτήσεις `PENDING_PROTOCOL` — έγκριση προϊσταμένου, χωρίς ακόμα πρωτόκολλο ΠΔΕΔΕ |
| **Σε Επεξεργασία** | Αιτήσεις `IN_REVIEW` και σχετικές |
| **Δικαιολογητικά** | Αιτήσεις `WAITING_FOR_DOCUMENTS` |
| **Απόφαση** | Προετοιμασία απόφασης |
| **Υπογραφές / ΣΗΔΕ** | `PENDING_SIGNATURES` — αναμονή ακριβούς αντιγράφου |
| **Ολοκληρωμένες** | `COMPLETED` |
| **Όλες** | Συνολική λίστα με φίλτρα |

Χρησιμοποιήστε τις καρτέλες για να βρίσκετε γρήγορα τις εκκρεμότητες — **αυτό είναι κεντρικό σημείο της καθημερινής σας δουλειάς** στο σύστημα.

## 2. Πλήρης ροή — κανονική άδεια (βασικό σενάριο trial)

Ακολουθήστε **μία** αίτηση από την αρχή μέχρι το τέλος, αλλάζοντας ρόλο (υπάλληλος → προϊστάμενος → χειριστής).

### Φάση 1: Πρωτόκολλο ΠΔΕΔΕ

1. Καρτέλα **Για Πρωτόκολλο** — βρείτε την αίτηση που εγκρίθηκε από προϊστάμενο.
2. Ανοίξτε τη **λεπτομέρεια**.
3. Καταχωρήστε **αριθμό πρωτοκόλλου ΠΔΕΔΕ** (δοκιμαστικός αριθμός για staging).
4. Αποθηκεύστε / επιβεβαιώστε.

**Αποτέλεσμα:** Η αίτηση περνά σε `Σε Επεξεργασία` (`IN_REVIEW`).

**Έλεγχος:**
- [ ] Η αίτηση μετακινήθηκε στη σωστή καρτέλα
- [ ] Το πρωτόκολλο εμφανίζεται στη λεπτομέρεια
- [ ] Οι διαθέσιμες ενέργειες άλλαξαν ανάλογα

### Φάση 2: Επεξεργασία (IN_REVIEW)

Από τη λεπτομέρεια, ανάλογα με τον τύπο άδειας:

- Ελέγξτε στοιχεία, ημέρες, υπόλοιπο αιτούντος
- Προχωρήστε σε **ετοιμασία απόφασης** όταν είστε έτοιμοι
- Ή ζητήστε **δικαιολογητικά** αν λείπουν (βλ. Φάση 2β)

**Έλεγχος:**
- [ ] Τα στοιχεία της αίτησης είναι πλήρη και σωστά
- [ ] Οι ενέργειες στην οθόνη ταιριάζουν με την κατάσταση

### Φάση 2β: Αίτηση δικαιολογητικών (προαιρετικό σενάριο)

1. Από `IN_REVIEW`, επιλέξτε **Αίτηση δικαιολογητικών** (ή ισοδύναμο).
2. Γράψτε σαφώς τι ζητάτε.
3. Η αίτηση περνά σε `WAITING_FOR_DOCUMENTS`.

**Συνέχεια ως υπάλληλος:**
- Ο υπάλληλος ανεβάζει/προσκομίζει τα δικαιολογητικά (ανάλογα με staging).

**Επιστροφή ως χειριστής:**
- Ελέγξτε τα αρχεία
- Επαναφέρετε σε `IN_REVIEW` ή προχωρήστε

**Έλεγχος:**
- [ ] Ειδοποίηση στον υπάλληλο
- [ ] Καρτέλα «Δικαιολογητικά» ενημερώνεται
- [ ] Μετά την παροχή, η ροή συνεχίζεται κανονικά

### Φάση 3: Ετοιμασία απόφασης

1. Από `IN_REVIEW`, επιλέγετε **Ετοιμασία απόφασης** / **Απόφαση**.
2. Συμπληρώνετε (όπου ζητείται):
   - Κείμενο απόφασης / πρότυπο
   - Στοιχεία κεφαλίδας, υπογράφοντα κ.λπ. (ανά staging)
3. Προεπισκόπηση PDF (αν διαθέσιμη).
4. **Δημιουργία τελικής απόφασης** (PDF).

**Αποτέλεσμα:** Κατάσταση προς υπογραφές (`PENDING_SIGNATURES` ή αντίστοιχη).

**Έλεγχος:**
- [ ] Το PDF δημιουργείται χωρίς σφάλμα
- [ ] Το περιεχόμενο είναι σωστό (όνομα, ημερομηνίες, ημέρες, πρωτόκολλο)
- [ ] Η αίτηση εμφανίζεται στην καρτέλα Υπογραφές/ΣΗΔΕ

### Φάση 4: ΣΗΔΕ — ακριβές αντίγραφο

Στην παραγωγή, η απόφαση περνά από ΣΗΔΕ για υπογραφές. Στο staging **προσομοιώνετε** το τελικό βήμα:

1. Ανοίξτε την αίτηση σε κατάσταση αναμονής υπογραφών.
2. Επιλέξτε **Ανέβασμα ακριβούς αντιγράφου** (ή ισοδύναμο).
3. Ανεβάστε δοκιμαστικό PDF (οποιοδήποτε έγκυρο αρχείο PDF για δοκιμή).
4. Επιβεβαιώστε **ολοκλήρωση**.

**Αποτέλεσμα:** `COMPLETED` — η άδεια θεωρείται ολοκληρωμένη.

**Έλεγχος:**
- [ ] Η κατάσταση γίνεται `Ολοκληρώθηκε`
- [ ] Το ακριβές αντίγραφο είναι προσβάσιμο για λήψη (όπου επιτρέπεται)
- [ ] **Υπόλοιπο υπαλλήλου** μειώθηκε (ελέγξτε ως υπάλληλος + ledger — βλ. `01-ypallilos.md`)

### Φάση 5: Μετά την ολοκλήρωση

- Η αίτηση εμφανίζεται στις **Ολοκληρωμένες**.
- Ο υπάλληλος βλέπει την ολοκληρωμένη αίτηση και ενημερωμένο υπόλοιπο.
- Στο **ledger** πρέπει να υπάρχει εγγραφή αφαίρεσης.

## 3. Άλλες ενέργειες χειριστή

### Απόρριψη από χειριστή

Από κατάλληλες καταστάσεις μπορείτε να **απορρίψετε** αίτηση (π.χ. `REJECTED_BY_LEAVES_DEPT`). Χρησιμοποιήστε **ξεχωριστή** δοκιμαστική αίτηση — συμπληρώστε λόγο.

### Κλείδωμα αίτησης (lock)

Όταν επεξεργάζεστε αίτηση, μπορεί να **κλειδώνει** για άλλους χειριστές (προστασία ταυτόχρονης επεξεργασίας). Αν «κολλήσει» κλείδωμα, σημειώστε χρόνο και αίτηση.

### Προβολή / λήψη συνημμένων

- Κρυπτογραφημένα δικαιολογητικά — ελέγξτε ότι ανοίγουν/κατεβαίνουν σωστά.
- Αν αποτυγχάνει η λήψη, καταγράψτε μήνυμα σφάλματος.

### Φίλτρα και αναζήτηση

Δοκιμάστε φίλτρα ανά κατάσταση, τμήμα, ημερομηνία — βοηθούν στην καθημερινή χρήση.

## 6. Αναφορά προβλημάτων (υποχρεωτική μορφή)

Αν εντοπίσετε **οτιδήποτε** λειτουργικό, οπτικό ή περιεχομενικό πρόβλημα:

1. **Μην** το αφήνετε ασαφές — γράψτε **αναλυτικά**.
2. Συμπληρώστε **ένα αρχείο Word** με τις παρατηρήσεις σας και στείλτε το στον υπεύθυνο της δοκιμής.

### Για κάθε παρατήρηση, γράψτε:

| Πεδίο | Τι να βάλετε |
|-------|--------------|
| **Τίτλος / σύντομη περιγραφή** | π.χ. «Δεν εμφανίζεται κουμπί πρωτοκόλλου» |
| **Βήμα προς αναπαραγωγή** | Ακριβής σειρά ενεργειών (1, 2, 3…) |
| **Ρόλος / λογαριασμός** | π.χ. Χειριστής — `apettas@sch.gr` |
| **Αίτηση** | ID ή αριθμός πρωτοκόλλου ή όνομα αιτούντος |
| **Κατάσταση αίτησης** | π.χ. `PENDING_PROTOCOL` |
| **Αναμενόμενο** | Τι περιμένατε να γίνει |
| **Πραγματικό** | Τι έγινε αντί αυτού |
| **Σημείο οθόνης** | Dashboard / Λεπτομέρεια / Καρτέλα «Για Πρωτόκολλο» κ.λπ. |
| **Browser & συσκευή** | π.χ. Chrome 125, Windows 11 |
| **Screenshot** | **Υποχρεωτικό** για κρίσιμα θέματα — ενσωματωμένο στο Word |

### Παράδειγμα καλής καταγραφής

> **Πρόβλημα:** Μετά την καταχώρηση πρωτοκόλλου, η αίτηση μένει στην καρτέλα «Για Πρωτόκολλο».  
> **Βήματα:** 1) Σύνδεση ως apettas@sch.gr 2) Άνοιγμα αίτησης #42 του Kotsonis 3) Πρωτόκολλο «12345/2025» 4) Αποθήκευση

### Προθεσμία και μορφή

- **Μορφή παραδοτέου:** αρχείο **Word** (`.docx`), ένα συγκεντρωτικό έγγραφο ανά χειριστή ή ομαδικά.

- Ξεχωρίστε **κρίσιμα** (μπλοκάρουν τη δουλειά) από **μικρά** (οπτικά, διατύπωση).

## 7. Επιπλέον ιδέες για αποτελεσματική δοκιμή

- **Χρονομέτρηση:** Σημειώστε αν κάποιο βήμα (π.χ. PDF) είναι αργό — χρήσιμο για βελτιστοποίηση.
- **Σύγκριση με τρέχουσα διαδικασία:** «Στο χαρτί κάνουμε Χ, εδώ λείπει / είναι διαφορετικό».
- **Κείμενα:** Σχόλια για ετικέτες, μηνύματα, ορολογία — αν κάτι δεν είναι κατανοητό για νέο χρήστη.
- **Edge cases:** Αίτηση που τελειώνει Παρασκευή, αργία εντός διαστήματος, πολλές περίοδοι.
- **Πολλαπλοί ρόλοι:** Δοκιμάστε εναλλαγή ρόλου στον ίδιο λογαριασμό (`korsianou@sch.gr`) — μπερδεύεται η πλοήγηση;
- **Ειδοποιήσεις:** Φτάνουν στον σωστό χρήστη τη σωστή στιγμή;
- **Εκτύπωση/PDF:** Εκτυπώνεται σωστά η απόφαση;
- **Μην χρησιμοποιείτε πραγματικά προσωπικά δεδομένα** στο staging — μόνο δοκιμαστικούς λογαριασμούς.
